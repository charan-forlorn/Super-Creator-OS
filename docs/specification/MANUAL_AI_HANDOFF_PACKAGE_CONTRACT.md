# Manual AI Handoff Package Contract (Stage 5.5)

## Purpose

Define the deterministic local file package an operator can use to manually
hand a prompt or result context to a target AI/runtime outside SCOS. The
package is generated by `scos/control_center/manual_handoff_package.py`.

## Non-goals

The handoff package is not an AI adapter, API client, browser automation
workflow, app automation workflow, clipboard integration, background worker,
database record, or dispatch mechanism. It does not send prompts, read results,
launch applications, execute shell commands, or call the network.

## Package Structure

`create_manual_handoff_package(...)` creates a deterministic folder under the
caller-supplied local `output_dir`:

```
handoff_<handoff_id>/
  prompt.md
  context_summary.md
  handoff_instructions.md
  handoff_manifest.json
```

`output_dir` may be `str` or `pathlib.Path`. URL-like paths are rejected. The
implementation resolves the package directory and verifies it stays inside the
provided output directory before writing.

## Generated Files

`prompt.md` contains:

- target agent
- packet title
- objective
- prompt body
- expected result format
- expected artifacts
- constraints

`context_summary.md` contains:

- context references
- artifact references when available
- routing reason when available
- source packet id
- source result packet id when available

`handoff_instructions.md` contains the deterministic operator steps:

1. Open the target AI/runtime manually.
2. Paste or provide the prompt manually.
3. Wait for the AI result.
4. Save the result back into SCOS as a ResultPacket.
5. Do not let the AI commit/push without operator approval.

`handoff_manifest.json` contains:

- handoff id
- source packet id
- source result packet id when available
- routing decision id when available
- target agent
- target runtime id
- handoff mode
- created_at
- generated file list
- SHA-256 checksums for generated files
- schema version

All files are UTF-8 with LF newlines.

## Deterministic Manifest Rules

`handoff_id` is derived from stable fields:

- source packet id
- source result packet id
- routing decision id
- target agent
- target runtime id
- handoff mode
- caller-supplied `created_at`

The manifest is serialized with sorted keys. File checksums are computed with
stdlib `hashlib.sha256` after the content files are written. No clock, random,
uuid, network, shell command, or clipboard state is used.

## Supported Handoff Modes

- `manual_clipboard`
- `manual_app`
- `manual_cli`
- `manual_review_only`

`manual_clipboard` means the operator may manually copy text outside SCOS. It
does not mean SCOS reads from or writes to the clipboard.

## Future Handoff

review and package records. Stage 5.7 may introduce richer adapter behavior.
Those future stages must still preserve this contract's safety boundary unless
